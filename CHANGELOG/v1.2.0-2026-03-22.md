# Changelog — v1.2.0

**Released**: 2026-03-22
**Type**: Security patch + reliability improvements

---

## What Changed

### 🔐 Security Fixes

#### Webhook Signature Verification — Now Correct and Timing-Safe
Previous versions computed the HMAC signature over a re-serialized version of the request body, which could differ from the original bytes and cause valid requests to be rejected or invalid ones accepted.

This release:
- Uses the **raw request bytes** (as originally sent) for HMAC computation, matching the server-side signing behavior exactly
- Replaces the previous Web Crypto API (`crypto.subtle`) implementation with Node.js built-in `crypto` — more reliable across all n8n deployment environments
- Switches from plain string comparison to **constant-time comparison** (`timingSafeEqual`) to prevent timing-based signature enumeration attacks

#### Credential URL Validation
The Base URL credential field now validates the URL before any HTTP request is made:
- Rejects non-HTTP/HTTPS protocols
- Rejects private/internal network addresses
- Provides a clear error message if the URL is invalid

#### Resource ID Validation
All ID parameters (Lead ID, Client ID, Webhook ID) are now validated before being used in API request URLs:
- Empty strings throw a clear error instead of silently hitting the wrong endpoint
- IDs containing path-traversal characters (`/`, `..`, `?`) are rejected
- Error messages indicate exactly which field failed and why

### ⚡ Reliability Improvements

- **Request timeout**: All outbound API requests now have a 30-second timeout. Previously, a slow or unresponsive API could cause n8n executions to hang indefinitely.
- **Per-item headers**: Request headers are now created fresh for each item in the loop, preventing any possible mutation bleed between items.
- **Item correlation (`pairedItem`)**: All output items now carry correct `pairedItem` references, enabling proper item tracing in multi-branch workflows and merge node operations.

### 🔌 Credential Testing
Added connection test support — clicking **"Test connection"** or **"Save"** on the Brixfit API credential now makes a live API call to verify the key is valid and shows a success or error message immediately.

---

## Upgrade Notes

- **No breaking changes** to workflow behavior for valid inputs
- Workflows using invalid or empty IDs in expressions will now surface a clear error instead of silently misbehaving — review any workflows that use dynamic ID expressions
- The Base URL field now rejects private/internal IPs — self-hosted instances must use a publicly reachable hostname
