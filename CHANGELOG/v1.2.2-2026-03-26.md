## What's New

### Webhook: Enable / Disable operation

The **Brixfit** node now has a fourth Webhook operation: **Enable / Disable**.

Previously the Webhook resource only supported `Get All`, `Create`, and `Delete`. The Brixfit API has always had `PATCH /webhooks/:id` to toggle `is_active`, but the node never exposed it.

**How to use:**

1. Set **Resource** → `Webhook`
2. Set **Operation** → `Enable / Disable`
3. Enter the **Webhook ID**
4. Toggle **Active** → `true` to re-enable, `false` to pause without deleting

Returns the updated webhook object.

**Use case:** Temporarily pause webhook deliveries during a deployment or maintenance window without losing the registration.
